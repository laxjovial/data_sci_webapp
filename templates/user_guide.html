User Guide

A step-by-step guide to using the Comprehensive Data Science Web Application.

Phase 1: Data Ingestion

This is the starting point of your analysis. The goal is to load your dataset into the application.

  1. Choose your source: Use the dropdown menu to select whether you want to "Upload from Device" or load "From a Web Link (URL)".
  2. If uploading from device:
    • Click the "Choose a File" button.
    • Select a CSV or Excel file from your computer.
  3. If using a web link:
    • Paste the URL into the input field.
    • The application supports direct links to CSV/Excel files, as well as links to files on GitHub and Google Drive. For Google Drive, ensure the file is shared so that "anyone with the link can view".
  4. Click the "Load Dataset" button. You will then be taken to the Data Viewer page to see a summary of your data.

Phase 2: Data Cleaning

Once your data is loaded, the next step is to clean it. This page provides several tools to fix common data quality issues. Use the tabs to navigate between different cleaning actions.

  • Handle Missing Values: Select one or more columns and choose a strategy (e.g., 'drop', 'fill with mean') to handle empty cells.
  • Convert Data Type: Change the type of a column, for example, from text to a number.
  • Remove Duplicates: Click the button to remove any identical rows in your dataset.
  • Standardize Text: Clean up text columns by making them all lowercase, uppercase, etc.
  • Handle Outliers: Manage extreme numerical values that could skew your results.
  • Correct Inconsistencies: Fix typos or merge similar categories (e.g., 'USA' and 'United States') by providing a Python dictionary of mappings.

After each action, the data preview at the top of the page will update to reflect your changes.

Phase 3: Exploratory Data Analysis (EDA)

This is where you get to explore your data visually. Creating plots helps you understand distributions, find relationships, and discover patterns.

  1. Select Analysis Type:
    • Univariate: To explore a single variable (e.g., the distribution of 'Age').
    • Bivariate: To compare two variables (e.g., 'Age' vs. 'Income').
    • Multivariate: To see the correlation between several numerical variables.
  2. Select Columns: Based on your choice, select the column(s) you want to analyze.
  3. Click "Generate Plot". The interactive visualization will appear at the bottom of the page.

Phase 4: Feature Engineering

This step involves creating new features or modifying existing ones to improve the performance of machine learning models.

  • Create Feature: Create a new column from two existing numeric columns using a mathematical operation.
  • Encode Columns: Convert categorical (text) columns into numbers. Several methods are available depending on the nature of your data.
  • Scale Features: Standardize the range of your numerical columns so that they are all on a similar scale.
  • Rename/Drop Columns: Organize your dataset by renaming columns or removing ones you don't need.

Phase 5 & 6: Model Building & Evaluation

This is the core of the predictive pipeline. Here you will train machine learning models and see how well they perform.

  1. Define Your Goal:
    • Select the **Problem Type** (Classification or Regression).
    • Choose the **Target Variable** you want the model to predict.
    • Set the **Test Set Size** (the percentage of data held back to test the model's performance).
  2. Select Your Tools:
    • Choose the **Feature Variables** the model should use to make its predictions.
    • Select one or more **Models** to run. The application will train all the models you select.
  3. Click "Build and Evaluate Models". You will be taken to a results page.
  4. On the results page, you will see a table comparing all the models. For classification tasks, you will also see a confusion matrix for each model.

Phase 7: Model Improvement (Tuning)

After building your models, you can try to improve the best one by tuning its settings (hyperparameters).

  1. On the Model Evaluation Results page, click the "Tune Model" button next to the model you want to improve.
  2. This will take you to the **Hyperparameter Tuning** page.
  3. The page provides a set of common hyperparameters for that model with default values to test.
  4. You can edit these values. Provide comma-separated numbers or text for the settings you want to try.
  5. Click "Run Tuning". The application will use GridSearchCV to find the best combination of settings.
  6. The results, including the best parameters and the performance of the tuned model, will be displayed on the page.
